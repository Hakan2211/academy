<Step title="Turning inputs into output" kind="explain">

A firm's **production function** describes how its inputs become output. In the **short run**, at least one input is fixed — usually the factory, the land or the heavy machinery, which cannot be changed quickly. The firm can only vary the flexible inputs, most obviously **labour**.

Two ideas measure how production responds:

- **Total product** — the total quantity of output produced with a given number of workers.
- **Marginal product** — the *extra* output added by hiring one more worker.

Marginal product is the one that drives decisions, because a firm choosing whether to hire is really asking: *how much more will this next worker produce?*

</Step>

<Step title="The law of diminishing returns" kind="interactive">

Add workers to a fixed factory and watch both numbers move.

<ProductionFunction />

Total product keeps climbing, but the **marginal product bars shrink** with each new worker. This is the **law of diminishing marginal returns**: as you add more of a variable input to a fixed one, the extra output each additional unit provides eventually falls.

It is not that later workers are lazy — they are identical. The problem is that they are sharing a *fixed* amount of equipment and floor space. The first worker has every machine to themselves. The fifth is queuing for a free machine. The tenth is mostly getting in the way. The fixed factory becomes the bottleneck.

</Step>

<Step title="Marginal product, marginal thinking" kind="interactive">

Marginal product is just the marginal-thinking idea applied to a factory: the value of *one more* unit of an input. Each extra worker adds a bit less than the last, exactly like each extra unit in any marginal decision.

<MarginalThinking />

Read these bars as a worker's contribution against their wage: a firm keeps hiring while the value of a worker's marginal product is at least their wage, and stops when the next worker would add less than they cost. Diminishing returns guarantees that point eventually arrives.

</Step>

<Step title="Short run vs long run" kind="explain">

The short run is not a fixed number of weeks — it is defined by what the firm *can change*. In the **short run**, at least one factor is fixed, so the firm is stuck with diminishing returns from crowding its fixed input.

In the **long run**, *every* input is variable. The firm can build a second factory, buy more machines, or rent more land. There is no fixed bottleneck to crowd, so the long run is the world of *scale* decisions rather than diminishing returns — the subject of this world's final lesson.

<Callout type="info">
Diminishing returns is a **short-run** phenomenon, caused by a fixed input. It is not the same as diseconomies of scale, which is a long-run problem of coordinating a too-large organisation. Keep the two ideas apart.
</Callout>

</Step>

<Step title="Recap" kind="recap">

- The **production function** links inputs to output; **total product** is the total made, **marginal product** is the output added by one more worker.
- The **law of diminishing marginal returns**: adding a variable input to a fixed one eventually makes each extra unit add less.
- The cause is a **fixed input** (e.g. the factory) becoming a bottleneck — not lazy workers.
- This is a **short-run** idea (one input fixed). In the **long run** every input is variable, which opens up questions of scale.

</Step>
