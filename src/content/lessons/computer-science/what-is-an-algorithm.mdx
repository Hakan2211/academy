<Lesson>

<Step title="A recipe for solving a problem" kind="explain">

An **algorithm** is a finite, precise, ordered list of steps that solves a problem or completes a task. That's it — no magic. A recipe is an algorithm. So are the directions to a friend's house, and the long-division method you learned in school.

The word honours the 9th-century mathematician **al-Khwārizmī**, but the idea is ancient: humans have always written down repeatable procedures so anyone (or any *thing*) can follow them and get the same result.

</Step>

<Step title="The four things every algorithm must be" kind="explain">

For a procedure to count as a proper algorithm, it should be:

- **Finite** — it eventually stops. "Keep adding 1 forever" is not an algorithm.
- **Precise** — each step is unambiguous. "Add a pinch of salt" is fine for a cook but too vague for a machine.
- **Ordered** — the steps run in a definite sequence; order changes the outcome.
- **Effective** — each step is something the follower can actually do.

<Callout type="info" title="Determinism">
Most algorithms are **deterministic**: the same input always produces the same output, by the same path. That predictability is exactly what makes them trustworthy — and testable.
</Callout>

</Step>

<Step title="Why precision matters for machines" kind="explain">

A human cook can improvise around a vague recipe. A computer cannot. It does *exactly* what the steps say — including the mistakes. An algorithm "find a tall person" means nothing to a machine; "scan the list, remember the tallest seen so far, report it at the end" is something it can execute flawlessly.

Turning a fuzzy human goal into steps this precise is the real skill. The rest of this world is about doing it *well* — because the same problem can be solved by many different algorithms, some far better than others.

</Step>

<Step title="Recap" kind="recap">

- An **algorithm** is a finite, precise, ordered set of steps that solves a problem.
- It must be **finite**, **precise**, **ordered**, and **effective**.
- Most are **deterministic**: same input, same output, same path — which makes them predictable and testable.
- Machines do *exactly* what the steps say, so precision is non-negotiable.
- The same problem can be solved by many algorithms — quality varies hugely.

</Step>
